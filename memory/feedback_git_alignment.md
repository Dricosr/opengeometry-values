---
name: Keep git docs aligned with code changes
description: When editing public API or definitions, update README, CONTRIBUTING, CHANGELOG in the same pass
type: feedback
---

When any constant, core function, or public API changes (added, removed, renamed), verify and update README.md, CONTRIBUTING.md, and CHANGELOG.md in the same pass.

**Why:** The user wants the repo to always look organized and consistent to public visitors.

**How to apply:** After every source change that affects the public surface, do a quick check on the git-facing docs before finishing the task.
